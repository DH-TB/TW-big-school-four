# 购物系统ReactJS Spring

###需求描述
商店里里里进行行行购物结算时根据客户的购物⻋车(Cart)中的商品(Item)和商
店正在进行行行的优惠活动(Promotion)进行行行结算和打印收据(Receipt)。
已知该商店正在对部分商品进行行行“买二二赠一一”的优惠活动。

###前端
我们需要实现一一个前端⻚页面面,可以十十分简单,但是要包含以下⻚页面面

一一个列列表,包含所有商品的信息(名称,单位,单价,优惠信息),每个item
可以加入入购物⻋车,并且可以选择加入入购物⻋车的商品数量量。
一一个购物⻋车⻚页面面,从列列表⻚页面面可以进入入购物⻋车⻚页面面,购物⻋车⻚页面面列列出所有你选
择的商品。

确认购买后,将购物⻋车内的信息以以下格式传到后端,格式(样例例):
[
'ITEM000001',
'ITEM000001',
'ITEM000001',
'ITEM000001',
'ITEM000001',
'ITEM000003-2',
'ITEM000005',
'ITEM000005',
'ITEM000005'
]
其中对'ITEM000003-2'来说,"-"之前的是标准的条形码,"-"之后的是数量量。 当我
们购买需要称量量的物品的时候,会由称量量的机器器生生成此类标签(Tag),收银机负
责识别生生成收据。 最后收到后台返回的发票信息,用用一一个发票⻚页面面来显示这张发
票。


###后端
我们需要一一个服务器器用用Java Spring写成,可以使用用数据库(例例如MySQL)来存储数
据,也可以使用用一一个数据类返回一一个固定数组。 收到前端传过来的购物信息后(上面面的id数组),后端经过一一系列列的计算,我们最终返回给前端一一个如下格式的发
票,清单内容(样例例):
***<没钱赚商店>收据***
名称:可口口可乐,数量量:3瓶,单价:3.00(元),小小计:6.00(元)
名称:羽羽毛毛球,数量量:5个,单价:1.00(元),小小计:4.00(元)
名称:苹果,数量量:2斤斤,单价:5.50(元),小小计:11.00(元)
----------------------
总计:21.00(元)
节省:4.00(元)
**********************
我们不不需要模拟付款功能,我们直接返回清单就好
